# Cardinal Architecture Specification — Volume IV: World Packages

Volume III defined what can exist in a Cardinal world. Volume IV answers *how worlds are
assembled*: how radically different simulations — a medieval kingdom, a modern city, an
ocean with no people in it, a colony ship, a world where the dead walk — are all the same
engine running different packages.

## Chapters

1. [The World Package](01_The_World_Package.md) — the package contract, manifest, identity discipline, and the extension seam
2. [Rules and Domain Selection](02_Rules_and_Domain_Selection.md) — the world's constitution: which domains run, and every tunable law
3. [Content](03_Content.md) — data packs, cross-reference integrity, modular composition, and the content pipeline
4. [World Generation](04_World_Generation.md) — the layer model, authored/procedural/simulated sources, and the expiry of generation's privilege
5. [Scenarios](05_Scenarios.md) — situations, casts, perspective without privilege, and endings as observations
6. [Saves, Versioning, and Migration](06_Saves_Versioning_Migration.md) — the version triangle, format and package migration, and the decade-old save
7. [Validation and Testing](07_Validation_and_Testing.md) — the five-layer validation stack, and determinism / conservation / liveness / phenomenological testing
8. [Five Worlds](08_Five_Worlds.md) — Thornwall, Meridian, Pelagia, Kepler Station, and the Sundered March as standing falsification targets
