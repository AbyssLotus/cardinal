# Cardinal Definitive Reference — Volume II: Constructing Reality

This volume translates Volume I's philosophy into engineering contracts: how reality is
represented, how it advances through time, how systems transform it, how information
diverges from it, and how it survives beyond a single process.

## Chapters

1. [Reality is Data](01_Reality_is_Data.md)
2. [Time](02_Time.md)
3. [Systems](03_Systems.md)
4. [Information](04_Information.md)
5. [Persistence](05_Persistence.md)
5.5. [The Cardinal Lexicon (Addendum)](05_5_The_Cardinal_Lexicon.md)

Each chapter ends with a set of engineering invariants that every implementation of Cardinal
SHALL preserve, regardless of language or optimization strategy. The addendum defines the
normative vocabulary — Reality, Fact, Entity, System, Tick, and related terms — that
Volumes I and II's architecture is built from.
